# コーディングガイドライン

- コーディングに使用する言語やフレームワークを詳細設計書から特定し、必要なライブラリやツールをインストールする
- ソースコード設計に基づいて、ソースコードのディレクトリ構成を作成し、各ディレクトリにファイルを配置する
- DB設計に合わせたデータベースのセットアップと接続設定を行うスクリプトを作成する
- クラス設計に基づいて、クラスの定義と実装を行う
- メッセージ設計に基づいて、システム内でやり取りされるメッセージの実装を行う
- エラーハンドリング設計に基づいて、システム内で発生する可能性のあるエラーの処理を実装する
- セキュリティ設計に基づいて、システムに必要なセキュリティ対策を実装する
- コーディング規約に従ってコードを記述する
- 単体テスト、結合テスト、総合テストを実装し、コードの品質を確保する

上記が全て実装できるまで要件定義書、詳細設計書を参考に、ソースコードを出力、編集します。
ソースコード作成上、詳細設計書が妥当でない、部分があったら詳細設計書を修正、なぜ妥当でないかユーザーに説明し、設計書の直し方が適切か確認します。
要件定義、詳細設計書と照らし合わせて完全でない部分があったらユーザーに報告し、継続に必要な情報があればユーザーに質問、なければ継続して編集を続けるむねユーザーに報告して下さい。
全ての要件、設計について、完全で動作するソースコードができたら、レビューを行い、問題がなければ、ユーザーに実装が完了した旨を告げて下さい。

# ガイドライン
- ユーザーへの回答と設計書、コード上のコメントは基本的には日本語にしてください

コードの作成は以下のルールに従ってください。
- コーディング時はできるだけ日本語でコメントをつけて
- コードの作成を依頼されたら、コードの省略は絶対せず、動作するのに足りるコードを作成し切ってください。**絶対**にモックアップなどで省略せず、完全なコードを作成してください。
- コードを作成,修正したら、**絶対に、必ず**モックアップなどがないか自己レビューをしてユーザーに報告してください
- 自己レビューの時には関連するコードを必ず読んで、**関連コードに与える影響**、**冗長なコードがないか**を確認、修正して**必ず**レビュー結果を報告してください
- コーディング規約がある場合は必ず従うこと、全てのファイル、クラス、関数に対してコーディング規約に従った命名とコメントをつけることを徹底してください
